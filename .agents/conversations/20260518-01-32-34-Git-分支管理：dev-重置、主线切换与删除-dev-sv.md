18: | 本地路径渲染 | `fs` + DOM | `url("file:///...")` | 不支持 `file:///` |
19: | 写外部路径 | `fs.writeFileSync()` | `require('fs/promises').writeFile()` | 只能通过 `/api/file/putFile` |
20: 
21: **`window.require('fs/promises')` — 关键突破**：
22: 
23: 思源桌面端是 Electron 应用，`nodeIntegration: true` 启用了 `window.require`。插件可使用标准的 `require('fs/promises')` 访问 Node.js 文件系统模块（commit `65cd7a9` 验证）。`window.fs` 是 Lute GopherJS 暴露的非标准接口，行为可能变化，不推荐使用。
24: 
25: **移动端限制**：
26: 
27: Siyuan 移动端 (Android/iOS/HarmonyOS) 是纯原生 WebView，不存在 Node.js 运行时，`window.require` 为 `undefined`，`file:///` 协议不可用。测试降级方式：
28: 
29: ```typescript
30: if (!(window as any).require) {
31:     showMessage("当前非桌面端，本地文件夹功能不可用");
32:     return;
33: }
34: ```
35: 
36: ### 1.2 源类型可用性矩阵
37: 
38: | 源类型 | desktop | mobile / browser / docker | 枚举方式 | 渲染方式 |
39: |--------|---------|--------------------------|---------|---------|
40: | `local` | **可用** | 不可用 | `require('fs/promises').readdir()` | `url("file:///...")` |
41: | `upload` | **可用** | **可用** | `readDir("data/public/...")` | `url("/public/...")` 或 `file:///` |
42: | `assets` | **可用** | **可用** | `readDir("data/assets/...")` | `url("/assets/...")` |
43: | `url` | **可用** | **可用** | HTTP 下载 | `url(...)` |
44: 
45: **结论**：desktop 端支持全部三种源（`local` 提供零上传体验，同时保留 upload/assets 兼容）；非 Electron 端两种源可用（upload/assets）。URL 方式合并到 upload 源的添加入口中。

94:   data/assets/                    ← 思源笔记所有附件存放处（可能数万文件）
95:     wallpaper/                    ← 用户手动创建，专门放壁纸
96:       sunset.png
97:       mountain.jpg
98:     travel-photos/                ← 用户选定作为图片源
99:       beach.png
100:       tokyo.jpg

106:     1. 在设置界面打开「assets 文件夹选取器」

111:     1. readDir("data/assets/wallpaper/") → 列出 wallpaper/ 下文件
112:     2. readDir("data/assets/travel-photos/") → 列出 travel-photos/ 下文件
113:     3. 按扩展名过滤图片/视频

115:     5. 不复制、不上传、零额外存储

119: - 用户笔记中已有的图片直接作为背景使用，避免一张图片占用双倍空间

121: - 不会加载整个 `assets/` 目录，避免性能问题

123: #### 2.3.2 图片源统一管理设计
124: 
125: 三种源类型，desktop 端全支持，非 Electron 端排除 `local`：
126: 
127: ```
128: desktop (Electron, require('fs/promises') 可用)    fallback (mobile/browser/docker)
129:   type='local'   ← ← 独占                    type='local'   ✗ 隐藏
130:   type='upload'  ← → 通用                    type='upload'  ← → 通用
131:   type='assets'  ← → 通用                    type='assets'  ← → 通用
132: ```
133: 
134: **URL 图片/视频合并到 upload 源**：用户输入 URL 后，插件下载文件（desktop:`fs/promises` / fallback:HTTP fetch），上传到 `upload` 源所在的 `data/public/` 目录。URL 下载得到的文件与其他上传文件在同一个随机池中无差别对待，不需要一个独立的 `url` 源类型。
135: 
136: ```
137: sourceManager
138: ├── sources: ImageSource[]            # 所有已注册的源
139: │
140: ├── detectPlatform() → 'desktop' | 'fallback'
141: 
142: ├── addSource(type, path, label)
143: │   ├── type='local'  → 用户输入本地文件夹绝对路径
144: │   ├── type='upload' → HTML <input> 文件选择器 / URL 下载入口
145: │   └── type='assets' → 文件夹树选取器 (readDir 遍历)
146: │
147: ├── removeSource(id)
148: 
149: ├── scanAll() → ImageItem[]          # 非 Electron 端自动跳过 local
150: │   ├── type='local'  → (desktop) require('fs/promises').readdir(path) → 过滤扩展名
151: │   ├── type='upload' → readDir("data/public/...") → 过滤扩展名
152: │   └── type='assets' → readDir("data/assets/<子文件夹>") → 过滤扩展名
153: │
154: └── pickRandom(excludeUrl)            # 从统一列表中随机选取
155: ```
156: 
157: **三种源的差异处理：**
158: 
159: | 维度 | local | upload | assets |
160: |------|--------|--------|----|
161: | 适用平台 | desktop only | 所有 | 所有 |
162: | 文件归属 | 用户自有目录，只读 | 插件目录，可增删 | 思源资产，只读 |
163: | 枚举方式 | `require('fs/promises').readdir()` | `readDir()` | `readDir()` |
164: | 渲染 URL | `file:///` | `/public/...` 或 `file:///` | `/assets/...` |
165: | 云同步 | 不涉及 | 仅 fallback 端 | 文件会同步 |
166: | 存储开销 | 零 | 占用 data/ 空间 | 零 |

170: ## 三、资源类型区分（图片 vs 视频）
171: 
172: ### 3.1 渲染模式自动切换
173: 
174: `bgRender.ts` 根据文件扩展名决策渲染方式：
175: 
176: ```
177: ImageItem
178: ├── type='image' → <canvas> 元素
179: │   └── CSS background-image: url(...)
180: │       + background-size: cover
181: │       + opacity / filter: blur()
182: │       + background-position: X% Y%
183: │
184: └── type='video' → <video> 元素
185:     └── <video src="..." loop muted autoplay>
186:         + opacity / filter: blur()
187:         + object-fit: cover
188:         + 不支持 position 偏移（CSS 限制）
189: ```
190: 
191: ### 3.2 类型判定策略
192: 
193: ```
194: 判定优先级（快路径优先）:
195: 1. 扩展名匹配已知列表（在设置中，可通过string自定义，并检查输入是否合法） → 直接分类
196:     图片: .png .jpg .jpeg .gif .webp .bmp .svg .avif
197:     视频: .mp4 .webm .ogg .mov .avi .mkv
198: 2. 无法判定 → 跳过，不加入随机池（在console.log中记录一下哪些文件被掠过了）
199: ```
200: 
201: **MIME 探测**：desktop 端可通过 `require('fs/promises').readFile()` 读取文件头判断真实类型，但开销较高且收益有限。扩展名判定在 desktop 和 fallback 端均可用，性能更好。
202: 
203: ### 3.3 统一池内混用
204: 
205: 图片和视频共享同一个随机池：
206: - `scanAll()` 返回 `ImageItem[]`，每个 item 带有 `type: 'image' | 'video'`
207: - `bgRender` 渲染时根据 type 选择 canvas 或 video 元素
208: - 随机选择不区分类型，允许图片和视频混用
209: - 用户可通过 UI 过滤「仅图片」「仅视频」「全部」
210: 
211: ### 3.4 视频特有处理
212: 
213: 视频的 CSS 能力受限（video 元素不支持 `background-*` 属性）：
214: 
215: | 特性 | 图片 (canvas) | 视频 (video) |
216: |------|-------------|------------|
217: | opacity | body opacity | video CSS opacity |
218: | blur | canvas CSS filter | video CSS filter |
219: | X/Y 偏移 | `background-position: X% Y%` | **不支持** |
220: | 尺寸模式 | `background-size: cover` | `object-fit: cover` |
221: | 自动播放 | N/A | autoplay (需 muted) |
222: | 循环 | N/A | loop |
223: | 性能 | 静态，低开销 | 持续解码，较高 |
224: 
225: **典型用途**：静态壁纸用图片，动态氛围用视频（如飘雪、星空、流水），两者可由同一个随机池管理。

229: ## 四、存储与同步方案
230: 
231: ### 4.1 核心原则
232: 
233: 文件资源不进入思源工作空间时，不会触发云同步。desktop 端 `local` 源直接引用外部目录，天然隔离。非 Electron 端 `upload` / `assets` 源仍在 `data/` 内，会同步但接受此行为。
234: 
235: ### 4.2 全部配置本地化
236: 
237: ```typescript
238: // setLocalStorageVal("siyuan-plugin-background-cover", { ... })
239: // 全部配置可 reset 时清除，替代旧版 configs.json 文件名语义
240: interface AppConfig {
241:   // 显示
242:   activate: boolean;
243:   opacity: number;            // 0.0 - 1.0
244:   blur: number;               // 0 - 10 (px)
245:   positionX: number;          // 0 - 100 (%) 全局默认
246:   positionY: number;
247: 
248:   // 图片源
249:   localFolders: string[];     // desktop: 外部文件夹绝对路径
250:   assetDirs: string[];        // data/assets/ 下用户勾选的子目录路径
251: 
252:   // upload 源不持久化 — 固定路径 data/public/siyuan-plugin-bgcover/，启动时直接 readDir 扫描
253: 
254:   // 自动刷新
255:   autoRefresh: boolean;
256:   autoRefreshTime: number;    // 分钟
257: 
258:   // 主题
259:   disabledThemes: { dark: string[]; light: string[] };
260: 
261:   // 每图片偏移覆盖（key = 完整渲染 URL，用 URL 而非 hash 因为不需要跨设备同步）
262:   imageOverrides: Record<string, {
263:     positionX?: number;
264:     positionY?: number;
265:   }>;
266: 
267:   // 运行时状态
268:   currentFile: string | null;
269: }
270: ```
271: 
272: **没有 `ImageSource` 接口**。upload 固定路径不需要持久化；local/assets 只是字符串数组，不再需要 `id`、`type`、`label`、`enabled` 等字段。
273: 
274: ### 4.3 sourceManager 扫描逻辑
275: 
276: ```
277: scanAll() 启动时：
278: 1. upload → 固定路径 /public/siyuan-plugin-bgcover/ → readDir → 过滤扩展名
279: 2. assetDirs → 遍历配置中的每个子目录 → readDir → 过滤扩展名 → 路径不可访问则跳过
280: 3. localFolders → desktop only, require('fs/promises').readdir → 过滤扩展名 → 路径不可访问则跳过
281: → 合并为统一 ImageItem[]
282: 
283: 文件增删时增量更新：
284: - 用户上传 / 删除文件 → 仅重新扫描 upload 目录
285: - 用户添加 / 移除 source → 重新扫描该 source 目录
286: ```
287: 
288: ### 4.4 per-image 偏移覆盖
289: 
290: key 使用完整渲染 URL（`file:///home/.../img.jpg` 或 `/assets/sub/img.png` 或 `/public/.../hash.jpg`）：
291: 
292: - 随机选中图片时 → 查 `imageOverrides[url]`
293: - 找到 → 用覆盖的 positionX/Y 渲染
294: - 未找到 → 用全局默认 positionX/Y
295: 
296: 不考虑跨设备迁移：系统重装后视为新设备重新配置。文件夹被移动后文件夹源入口在 UI 中显示不可访问，用户删除重建即可。不需要 hash 多一层间接。
297: 
298: ### 4.5 为何移除 `sizeMode`
299: 
300: `cover` 即可满足绝大多数需求：
301: - 图片自动缩放填满屏幕，不变形（保持宽高比）
302: - `repeat` 大多重复背景视觉效果奇怪
303: - `center`/`contain` 边缘容易留白
304: 
305: 不提供选择，简化代码和 UI。
306: 
307: ### 4.6 为何不需要 `prevTheme`
308: 
309: - 背景层是独立的 canvas/video 元素，不受主题 CSS 影响
310: - 主题变化时只需检查 `disabledThemes` 列表：如果新主题在禁用列表中，隐藏背景；否则显示
311: - 用 `MutationObserver` 监听 `<html>` 属性变化实时检测，无需持久化
312: 
313: ### 4.7 文件资源同步
314: 
315: | 源类型 | 文件位置 | 云同步 | 说明 |
316: |--------|---------|--------|------|
317: | `local` | 用户指定绝对路径 | 不涉及 | 完全在思源工作空间之外 |
318: | `assets` | `data/assets/<子文件夹>` | 会同步 | 本来就是笔记附件，同步合理 |
319: | `upload` | `data/public/siyuan-plugin-bgcover/` | 会同步 | 含 URL 下载的图片，可手动加 syncignore |

326: | `/api/storage/getLocalStorage` | `/api/storage/getLocalStorageVal()` |
